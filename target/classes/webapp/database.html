🗂 表结构详情

filings 表 (13F文件信息)

用途: 存储13F文件的基本信息

主键: id (自增)

唯一约束: accession_number

字段名 数据类型 约束 说明
id INTEGER PRIMARY KEY AUTOINCREMENT 自增主键
cik TEXT NOT NULL 机构CIK编号 (10位数字)
company_name TEXT NOT NULL 机构名称
filing_type TEXT NOT NULL 文件类型 (如13F-HR, 13F-HR/A)
filing_date DATE NOT NULL 报告日期 (季度末)
accession_number TEXT NOT NULL UNIQUE SEC文件编号 (格式: 0000000000-00-000000)

holdings 表 (持仓明细)

用途: 存储每个13F文件中的具体持仓信息

主键: id (自增)

外键: filing_id → filings.id (CASCADE DELETE)

字段名 数据类型 约束 说明
id INTEGER PRIMARY KEY AUTOINCREMENT 自增主键
filing_id INTEGER NOT NULL, FOREIGN KEY 关联的13F文件ID
name_of_issuer TEXT NOT NULL 发行人名称 (如Apple Inc)
cusip TEXT NOT NULL CUSIP标识符 (9位字母数字)
value DECIMAL(15,2) 允许NULL 持仓市值 (美元)
shares BIGINT 允许NULL 持股数量

🔗 表关系

一对多关系: 一个filing可以包含多个holdings

级联删除: 删除filing时自动删除相关的holdings记录

数据完整性: 外键约束确保数据一致性

📝 示例查询

查看某机构的最新持仓:

SELECT h.name_of_issuer, h.cusip, h.value, h.shares FROM holdings h JOIN filings f ON h.filing_id = f.id WHERE f.cik = '0001524258' ORDER BY f.filing_date DESC, h.value DESC

查看持仓总市值排名:

SELECT f.company_name, SUM(h.value) as total_value FROM filings f JOIN holdings h ON f.id = h.filing_id GROUP BY f.cik, f.company_name ORDER BY total_value DESC
